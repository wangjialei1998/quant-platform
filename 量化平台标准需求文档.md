# 量化平台 MVP 标准需求文档

## 1. 项目概述

本项目面向个人量化投资使用场景，建设一个支持 A 股股票与场内 ETF 的策略回测、模拟盘组合监控、交易信号分析和邮件提醒的量化平台。

第一版聚焦日线级别策略闭环：用户上传 Python 策略，创建组合并选择标的，系统基于历史行情完成回测，从创建后进入自动模拟监控，并在产生信号、成交或运行异常时发送邮件提醒。

第一版不接入券商真实交易，不支持多用户权限，不实现分钟级回测，不强制处理除权除息复权数据。

## 2. 产品范围

### 2.1 第一版范围

- 支持个人单用户使用。
- 支持 A 股股票与场内 ETF 作为交易标的。
- 支持指数作为收益曲线对比基准，例如沪深 300。
- 支持 Python 策略上传、编辑、测试、查看和管理。
- 策略规范采用 `backtrader.Strategy` 类。
- 支持日线级历史回测。
- 支持组合创建后进入模拟盘监控。
- 支持单策略、多标的组合。
- 支持定时运行与手动触发更新。
- 支持组合暂停、恢复、删除和查看。
- 支持设置邮件提醒开关。
- 支持信号、成交、错误三类邮件提醒。
- 支持行情数据落库与缓存删除管理。
- 支持核心绩效指标、收益曲线、买卖点标注、历史交易、资金流水、持仓变化和信号洞察分析。

### 2.2 暂不纳入第一版

- 分钟线级别回测与监控。
- 真实券商下单。
- 多用户、登录权限、团队协作。
- 完整复权体系。
- 复杂 A 股差异化规则，例如 ST、科创板、创业板、北交所细分涨跌幅规则。
- 分红、送转、配股、复权收益精确还原。
- 多策略组合与策略权重分配。

## 3. 核心业务规则

### 3.1 策略规则

用户可以通过页面代码编辑器或文件上传方式创建策略。

策略必须是 Python 文件，并包含可运行的 `backtrader.Strategy` 子类。新增或修改策略时，系统需要执行语法检查和最小化运行测试。如果策略存在语法错误、运行错误或接口不符合规范，页面应返回错误信息和运行日志。

策略管理功能包括：

- 新增策略
- 编辑策略代码
- 上传策略文件
- 删除策略
- 查看策略详情
- 查看策略描述
- 查看测试状态和错误日志

策略运行需要在隔离环境中执行，并设置超时时间，避免死循环、长期阻塞或异常占用资源。

### 3.2 组合规则

组合由一个策略、多个标的、起始日期和初始资金组成。

创建组合后，系统需要先从起始日期回测到当前最近交易日，生成初始净值、持仓、交易记录、资金流水和指标数据。组合创建成功后默认进入模拟盘监控状态。

组合支持：

- 创建组合
- 查看组合列表
- 查看组合详情
- 暂停监控
- 恢复监控
- 手动触发一次更新
- 删除组合
- 设置是否启用邮件提醒

组合第一版采用单策略多标的模式，不支持一个组合绑定多个策略。

### 3.3 日线运行规则

日线监控支持自动定时运行和手动运行。

默认运行逻辑：

- 交易日收盘后拉取或更新当日行情。
- 策略基于最新可用日线数据生成买卖信号。
- 买卖信号生成后发送邮件提醒。
- 实际模拟成交按“次一交易日开盘价成交”处理。
- 成交完成后更新持仓、资金流水、交易记录和组合指标。
- 成交完成后发送邮件提醒。
- 策略运行失败、行情更新失败、邮件发送失败等异常需要记录日志；其中策略运行失败和行情更新失败需要触发错误提醒。

页面需要提供手动触发更新能力，用于用户主动刷新组合状态。

### 3.4 交易规则

第一版支持基础 A 股交易规则：

- 买入后 T+1 才允许卖出。
- 股票和 ETF 按 100 股整数手交易。
- 支持手续费、印花税和滑点参数。
- 买入时校验可用资金。
- 卖出时校验可卖持仓。
- 停牌数据缺失时不成交。
- 涨停时默认买入不可成交。
- 跌停时默认卖出不可成交。
- 行情免费接口阶段使用不复权价格。
- 除权除息造成的历史收益偏差作为第一版已知限制。
- 后续切换 TickFlow 付费接口后，再扩展复权因子和前复权/后复权能力。

## 4. 功能需求

### 4.1 策略管理

策略管理页以列表形式展示所有策略，包含策略名称、描述、创建时间、更新时间、最近测试状态和操作入口。

新增策略时支持两种方式：

- 页面内代码编辑。
- 上传 Python 文件。

保存策略时系统需要：

- 校验 Python 语法。
- 校验是否符合 Backtrader 策略接口。
- 使用样例行情执行一次最小化测试。
- 返回测试结果、错误堆栈和日志。

策略详情页展示：

- 策略名称
- 策略描述
- 策略代码
- 最近测试结果
- 最近运行日志
- 使用该策略的组合列表

### 4.2 组合管理

组合管理页以卡片或列表形式展示组合。

组合卡片展示：

- 组合名称
- 绑定策略
- 标的数量
- 初始资金
- 当前总资产
- 最新净值
- 当前收益率
- 最大回撤
- 最新更新时间
- 运行状态

创建组合时需要填写：

- 组合名称
- 策略
- 标的列表
- 初始资金
- 起始日期
- 邮件提醒开关
- 手续费、滑点等交易参数，未填写时使用系统默认值

组合创建后系统自动执行从起始日期到当前日期的历史回测。

### 4.3 组合详情

组合详情页展示组合绩效、收益曲线、持仓、交易和资金流水。

核心指标包括：

- 最新净值
- 年复合收益率
- 胜率
- 盈亏比
- 夏普比率
- 当前回撤
- 最大回撤
- 最大回撤时间
- 波动率
- SQN
- VWR
- 总交易次数
- 运行天数
- 建仓日期
- 更新日期

图表包括：

- 组合净值曲线
- 收益曲线与基准对比
- 回撤曲线
- 买卖点标注
- 月度收益贡献
- 年度收益贡献
- 每年表现
- 每月表现
- 历史持仓金额变化

图表要求：

- 使用 ECharts。
- 支持日期范围缩放。
- 支持局部放大。
- 支持鼠标悬停查看数据。
- 买点和卖点需要在图中明确标注。
- 可选择对比基准，例如沪深 300。

列表包括：

- 历史交易记录：日期、代码、名称、方向、数量、价格、金额、手续费、成交状态。
- 资金流水：日期、类型、变动金额、可用资金、持仓市值、总资产。
- 历史持仓：日期、代码、名称、持仓数量、持仓市值、权重。

### 4.4 信号洞察

信号洞察页用于分析组合策略行为。

核心内容包括：

- 标的历史价格走势。
- 买卖信号标注。
- 是否显示交易信号的开关。
- 是否标准化价格的开关。
- 多标的价格标准化到 0-10 区间后展示。
- 空仓和持仓天数占比。
- 买入信号有效性。
- 卖出信号有效性。
- 5 个交易日和 20 个交易日后的信号表现。
- 总交易次数。
- 平均交易间隔。
- 买入信号次数和平均间隔。
- 卖出信号次数和平均间隔。
- 风险信号列表。
- 月度波动率分析。

风险信号第一版包括：

- 30 天内交易次数过多。
- 买入后短期下跌次数过多。
- 短期内 B-S-B 频繁切换。
- 长期空仓。
- 单标的波动率异常。

### 4.5 行情数据与缓存管理

系统需要将已获取的行情数据保存到数据库中。

行情数据使用原则：

- 回测时优先读取数据库已有数据。
- 数据库缺失时调用 TickFlow 免费接口获取。
- 获取成功后写入数据库。
- 组合创建后，相关标的从组合起始日期开始的数据需要保存。
- 后续交易日增量更新。
- Redis 可用于热点数据缓存和计算结果缓存。

缓存管理页支持：

- 按标的查询已有数据范围。
- 查看某标的某时间段内是否存在数据。
- 删除指定标的、指定日期范围的数据。
- 删除后再次回测时重新拉取行情。

### 4.6 邮件提醒

第一版支持 SMTP 邮件配置。

邮件触发事件包括：

- 策略产生买入或卖出信号。
- 模拟成交成功。
- 策略运行失败。
- 行情数据更新失败。
- 定时任务执行异常。

无信号、无成交时默认不发送邮件。

邮件内容应包含：

- 组合名称
- 策略名称
- 标的代码和名称
- 信号方向或成交方向
- 信号日期或成交日期
- 价格、数量、金额
- 当前组合总资产
- 错误摘要和日志入口，适用于异常提醒

## 5. 技术与架构要求

前端技术栈：

- Vue
- ECharts
- 支持响应式布局
- 图表支持缩放、拖拽和 tooltip

后端技术栈：

- Python
- FastAPI
- Backtrader
- Docker
- TickFlow 行情接口
- Redis 缓存
- 数据库存储行情、组合、交易、资金流水、指标和运行日志

后端模块建议按功能拆分：

- 策略管理模块
- 回测引擎模块
- 组合管理模块
- 行情数据模块
- 交易规则模块
- 指标计算模块
- 信号洞察模块
- 定时任务模块
- 邮件通知模块
- 缓存管理模块
- 系统日志模块

后端代码需要模块化、结构清晰、保留必要中文注释，并关注性能。

## 6. 非功能需求

性能要求：

- 已缓存行情数据不得重复调用外部 API。
- 同一组合的常用图表数据可以预计算并落库。
- 组合详情页应优先读取已计算结果。
- 大量历史数据图表需要分页或按时间范围加载。

可靠性要求：

- 定时任务失败需要记录日志。
- 策略运行失败不得影响其他组合。
- 邮件发送失败不得中断组合计算。
- 行情接口失败需要有明确错误状态。

安全要求：

- 用户上传策略代码必须隔离执行。
- 策略运行需要超时限制。
- 策略运行日志需要可追踪。
- 第一版个人单用户场景下不强制做复杂权限，但后台接口仍需避免危险文件访问。

可维护性要求：

- 指标计算逻辑与页面展示分离。
- 行情数据获取与行情数据存储分离。
- 交易规则独立封装，便于后续扩展分钟线和真实交易。
- 复权逻辑预留扩展位置，但第一版不实现完整复权。

## 7. 验收标准

第一版完成后应满足以下验收条件：

- 可以上传一个符合 Backtrader 规范的 Python 策略。
- 策略有语法错误时，页面能展示明确错误信息。
- 可以创建一个股票或 ETF 组合，绑定一个策略和多个标的。
- 组合创建后能从起始日期回测到当前最近交易日。
- 回测结果能生成净值、收益、回撤、交易记录、资金流水和持仓记录。
- 组合能在定时任务中自动更新。
- 用户可以手动触发组合更新。
- 用户可以暂停和恢复组合监控。
- 买卖信号能在图表中标注。
- 收益曲线可以选择基准对比。
- 图表支持日期范围缩放和局部放大。
- 信号洞察页能展示信号分布、信号有效性、交易频率、风险信号和波动率分析。
- 产生信号、成交或错误时能发送邮件。
- 行情数据能落库，重复回测优先使用数据库数据。
- 缓存管理页可以查询和删除指定标的、指定时间段的数据。
- 免费行情阶段按不复权价格计算，并在页面或说明中明确该限制。

## 8. 默认假设

- 第一版面向个人本地或私有部署使用。
- 第一版只做模拟盘，不进行真实交易。
- 第一版只支持日线。
- 第一版策略接口采用 Backtrader。
- 第一版组合模式为单策略多标的。
- 第一版交易信号按收盘后生成，模拟成交按下一交易日开盘价。
- 第一版行情使用 TickFlow 免费接口，不复权。
- 指数仅作为对比基准，不作为可交易标的。
- 股票和 ETF 是第一版可交易标的。
- 邮件提醒覆盖信号、成交和错误。
- 除权除息、分钟线、真实下单、多用户权限作为后续版本扩展。
